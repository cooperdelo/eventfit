## **EventFit Architecture Diagram – Messaging + Events + Outfits + Calendar**

### **1. Core Modules (Boxes in Diagram)**

1. **User Module**
   - Profile
   - Closet / Rentable Items
   - Friend List
   - Verified `.edu` email
   - Settings
2. **Event Module**
   - Event Detail
   - Event Feed / Outfit Feed
   - Public / Private toggle
   - Event Calendar Integration
   - RSVP tracking (optional future)
3. **Organization Module**
   - Org Dashboard
   - Member List / Roles
   - Org Events
   - Org Chat
4. **Outfit Module**
   - Outfit Posting / Details
   - Outfit Images, Tags (Size, Brand, Rentable)
   - Outfit Linking to Events / Chats
   - Outfit Reminders
5. **Chat / Messaging Module**
   - Event Chat
   - Org Chat
   - Direct Messaging (Phase 2)
   - Attachments / Outfit Links
   - Notifications Integration
6. **Notification Module**
   - Push / In-App Notifications
   - Event Updates
   - Chat Mentions
   - Rental Status Alerts
   - Outfit Reminders
7. **Renting & Payment Module**
   - Stripe Connect Escrow
   - Rent Request / Approve / Return
   - Penalty Fees / Dispute Handling
   - Payment Release Logic
8. **Admin Module**
   - User / Org / Event Management
   - Content Moderation
   - Analytics & Reporting

---

### **2. Relationships / Connections (Arrows in Diagram)**

- **User → Event Module**
  - Joins event (public/private)
  - Posts outfits to event feed
  - Receives notifications for event activity
- **User → Organization Module**
  - Member of org
  - Posts outfits / messages in org chat
  - Views org events
- **Event Module ↔ Outfit Module**
  - Outfits tagged to events
  - Outfit feed displayed in event detail
- **Event Module ↔ Chat Module**
  - Event chat lives inside event detail
  - Chat messages can link outfits
- **Organization Module ↔ Chat Module**
  - Org chat connects to org dashboard
  - Messages can reference org events
- **Chat Module ↔ Notification Module**
  - New messages trigger notifications
  - Mentions trigger priority notifications
- **Outfit Module ↔ Notification Module**
  - Outfit reminders
  - Rental updates (approved, returned, overdue)
- **Renting & Payment Module ↔ User / Outfit / Event**
  - Users request rental → funds held in escrow
  - Rental linked to outfit and user profiles
  - Payment released after return confirmation
- **Admin Module ↔ All**
  - Admin oversees events, orgs, users, rentals, disputes, and chats

---

### **3. Data Flow Highlights**

1. **User posts outfit → updates Event Feed → triggers Notification**
2. **User sends chat message → updates Event/Org Chat → triggers Notification**
3. **Rental requested → funds held in Escrow → return confirmed → payout released**
4. **Event creation → linked to Org → calendar integration → participants notified**
5. **Mentions / outfit attachments → live preview in chat + feed updates**

---

- **Profile / Closet**

### **5. Backend Services (indicate as layer below modules)**

- **Authentication**: Firebase Auth or similar
- **Database**: PostgreSQL / Supabase
  - Users, Outfits, Events, Orgs, Chats, Rentals
- **Realtime Messaging**: WebSocket / Supabase Realtime
- **Payments**: Stripe Connect Escrow
- **Notifications**: FCM / APNs / In-app
- **Admin APIs**: REST endpoints for management
- **File Storage**: Cloud Storage (Images, Outfit Photos, Attachments)
